Resources

Compilers

You'll need a Haskell Compiler. Almost all of the other implementations are dead or abandoned now (If you're interested, Hugs, yhc and nhc98 are the dead compilers).

Books

There are many, many books on Haskell. This course will be academically driven, so I'll only recommend academically driven books for this course. However, there are other books that are more suitable to using Haskell in the "Real World".

  • Programming Haskell by Graham Hutton
  • The Haskell Road to Logic, Maths and Programming by Jan van Eijck
  • Haskell: The craft of functional programming by Simon Thompson

The first one is an excellent book, the second is also excellent, but more difficult to read, and the third and final book is easier to read, but not (in my opinion) as high a standard as the first two.

Tips and Hints

There are many things you can do to increase the benefit of the exercises, here are just a few:

  • Don't jump in with a higher-order function, especially for harder exercises. Try simpler techniques first.
  • Pay attention to the types.
  • Just because a proof isn't requested, doesn't mean that you should avoid them.
  • QuickCheck is your friend. Beats the hell out of a single test file or manually typing test cases. If people think this would be useful, I may start producing a set QuickCheck tests for the exercies, and changing up the release cycle a little bit.
  • Just because you've done one exercise, doesn't mean it's done. You can likely think up several implementations, spanning primitive recursion, recursion with an accumulator, and functions expressed as a higher order function. Try to re-express your function in different ways. Experiment to determine which performs better, and why.
  • Use a version control system and commit often. This means that if you screw up, but had a half way decent solution, you can easily recover it. Some common ones are Darcs, Git and Mercurial. Most Haskell projects traditionally use Darcs, so if you're planning to use a lot of Haskell, it may be worthwhile familiarising yourself with it.

Errata

Due to differences in different Haskell versions (Specifically Haskell98 vs. Haskell2010) the exercises.hs files may have some errors.

Exercises 1

Due to Ord being removed from Num in Haskell2010, the types of even' and odd' when using GHC7 should be "even' :: (RealFrac a) => a -> a -> a" and "odd' :: (RealFrac a) => a -> a -> a".

This is because (in Haskell2010) Num no longer implies that a value may be orderable. This was decided by the Great Haskell Cockup-committee.
